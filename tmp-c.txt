feat(R422): Save snapshot toast offers one-click Share

The POST /api/workspaces response already returns the freshly-saved
CloudWorkspaceRecord with its id, but we were discarding it. After a
successful save we capture the id and attach a Share action to the
success toast. Clicking it routes through the existing
toggleShare(summary) path (confirm dialog -> enable public -> copy
URL), so users who saved specifically in order to share get there in
one click instead of scanning the new row for the icon. Dates fall
back to new Date().toISOString() if the server omits them.
