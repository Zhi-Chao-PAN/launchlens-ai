feat(R423): roving tabindex on hypothesis cards for AT navigation

Every hypothesis article was hard-coded tabIndex={0}, which meant
Tab walked through every card instead of stopping at the active
one (standard listbox/grid behaviour). Combined with the Arrow key
handler added in earlier rounds, tab users ended up with a
double-pain: Tab visited every card, and Arrow keys also moved
focus but left all cards tabbable.

Added a focusedExperimentId ref via useState. The cards now compute
tabIndex from it (first card defaults to 0 before any focus,
focused card stays 0, all others -1) and an onFocus handler tracks
which one currently owns focus. Pressing Arrow/Home/End still moves
focus imperatively and the next render moves the tab stop along
with it. After Tab into the list, screen-reader and keyboard users
can now traverse via arrows with a single Tab stop per list,
matching WAI-ARIA listbox pattern.

aria-posinset/setsize were tried but role=group does not expose
them; the existing aria-label already includes "Hypothesis i/total"
so that information is still announced.
